# Project History

One file per session, named `YYYY-MM-DD.md` (multiple sessions in a day are appended as
`## Session N` sections). Written by the `/beer` command at the end of each session.

Each file records:
- Features shipped or meaningfully progressed
- Key decisions made and why
- Gotchas discovered
- DB/infra changes applied (migrations run, seed/compose/nginx changes)
- What's up next

Use the `/history` command to query across these files.
