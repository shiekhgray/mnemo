### Context

### Shipped — Drag to Reorganize (`prd/drag-reorg.prd`), all 4 phases

Lifts the read-only restriction on the Wall tab **for container positions only** (fixtures
stay read-only — that's `layout-editor.prd`). From the wall you can now reorganize by touch.

- **Backend** (`positions.swap_slot()` + `POST /containers/{id}/swap-slot`): a new *atomic*
  swap — A takes B's slot and B takes A's, neither benched. Distinct from `assign_slot()`,
  which *bumps* the occupant to benched. Stages through a `flush()` to dodge the unique
  `slot_id` constraint mid-swap; an optional `expected_source_slot_id` rejects a swap computed
  against a stale board (409). Degrades to a plain move if the target turns out empty.
- **Drag layer** (`web/src/dnd/DragContext.jsx`): pointer-events (not HTML5) so finger == mouse.
  **Long-press to lift** (350 ms) discriminates tap (opens) / plain-drag (scrolls) / held-drag
  (moves). Floating ghost, `elementFromPoint` hit-testing of `data-drop-*` targets with the
  highlight toggled directly on the DOM node (kept out of React's render path for a 64-cell
  cabinet), optimistic `['bins']` update + rollback + **Undo toast** (incl. inverse swap/assign).
- **Bench sidebar** (`web/src/components/BenchSidebar.jsx`): collapsible, **every page** (lives
  in `Layout.jsx`), reactive dock — left-edge drawer when landscape, bottom sheet when portrait
  (matchMedia on aspect-ratio). Both a drop target (drag a drawer here → bench) and a drag
  source (drag a chip onto a drawer → place/swap). Auto-peeks open while dragging. It's the
  **bridge for cross-cabinet moves** (only one cabinet is on screen at a time).
- **Wall cells** (`LocationsPage.jsx` `CabinetDetail`): occupied cells became drag sources, all
  cells became drop targets carrying `data-drop-id`/`-kind`/`data-slot-id`/`data-occupant-*`.
  Split into `OccupiedCell`/`EmptyCell` so `useDraggable` stays an unconditional hook.
- **`/benched` page retired** and replaced by the sidebar (route now redirects to `/locations`,
  nav link removed, `BenchedPage.jsx` deleted).

### Drop outcomes (the table the code implements)
placed→empty = **move** (reuse `assign-slot`); placed→occupied = **swap** (new endpoint);
placed→bench = **bench**; bench-chip→empty = **place** (assign); bench-chip→occupied = **bump**
(assign, occupant benched — *not* a swap, since the chip has no slot to trade).

### Key decisions
- **Swap is a real new operation**, not reused assign — drag-onto-occupied "trades places";
  a silent bump-to-benched would be a surprise. (Bump semantics are kept only for a *benched
  chip* landing on an occupied drawer, where there's no source slot to swap.)
- **Recover from the working tree, not the old session.** The half-done files were already on
  disk and `git status` mapped cleanly to the PRD's phases, so resuming in a fresh session was
  faster than `claude --resume`-ing the capped one.

### Gotchas
- The **api no-bind-mount** gotcha bit again as expected: the swap endpoint existed in source
  but not in the running container until `docker compose up -d --build api`. Verified live via
  `/openapi.json` afterwards.
- ESLint (newer rules): non-component hook exports in a context file trip
  `react-refresh/only-export-components` — matched the repo convention (`AuthContext`) and added
  `// eslint-disable-next-line` before `useDrag`/`useDraggable`. A `setState` resync in a
  `matchMedia` effect trips `react-hooks/set-state-in-effect` — disabled inline with a why.

### DB / infra changes
- **None** — pure position movement over the existing schema, no migration. Rebuilt the `api`
  container to deploy the new endpoint.

## Session 3 — Storage layout editor frontend (all 3 phases), recovered from a crash

### Context
Recovered a crashed SSH session via `/coffee`: the working tree was clean and `git log` ran

### Shipped — all phases of `prd/layout-editor.prd`
- **Phase 1 — cabinet CRUD with presets** (`d23faf7`). New `web/src/components/CabinetEditModal.jsx`
  + an **"Edit layout"** toggle on the Wall tab (read-only finder stays default). Empty wall cells

  to place/move (swap on collision), delete. Empty instance shows "+ Add your first cabinet" — the
  bootstrapping goal. Destructive reshapes/deletes confirm their blast radius (occupied drawers
  benched, never deleted), computed **client-side** from the same band→address scheme as
  `positions.py` so it's a confirm-*before*-apply.
- **Phase 2 — custom band grids** (`ad5dcfc`). A 4th **"custom"** quick-pick opens a band editor
  (per-band cols×rows, add/remove), seeded from the current grid, with a WYSIWYG live preview.
  Saves `grid_spec` instead of a preset name. Generalized row heights off the band model
  (`LocationsPage.rowWeights`).
- **Phase 3 — chest CRUD** (`5f93c6a`). New `web/src/components/ChestEditModal.jsx` (name + drawer
  count, each drawer a front+back box). Tackle tab built out from a stub: a read-only finder
  rendering each chest's drawers as front/back box cells (occupied → link to container, empty →
  create one, mirroring the wall) plus an Edit-layout mode (add / tap-to-edit / delete). Shrinking
  benches the occupied boxes of removed drawers.

### Key decisions
- **Generalized `rowFlex` = "each band is an equal vertical slice" (1/numBands), split evenly among
  its rows.** This was the non-obvious bit: weighting bands by row count would render half-half as
  4/7 over 3/7, breaking the preset. Equal-share-per-band is the *only* clean rule that reproduces
  all three Akro-Mils presets exactly (single band → uniform 1/8 or 1/6; half-half → two 1/2 slices,
  4 narrow rows over 3 wide) *and* renders arbitrary custom grids sensibly. Applied identically in
  `MiniCabinet`, `CabinetDetail`, and the modals' `GridPreview` so previews are WYSIWYG.
- **Confirm-before-apply, client-side.** The backend reconcile returns `{added,removed,bumped}`
  *after* applying; to warn first, the frontend recomputes the blast radius from the band→address
  scheme (cabinets) / `drawer_number > count` (chests). Verified the two agree via live round-trips.
- **Chests reuse the wall's drawer-cell behavior** (tap empty box → create container, tap full →
  open) for consistency; chests are *not* in the drag-reorg layer (that's wall-only for now).
- **Committed to `main` directly**, matching this repo's established single-user workflow (backend
  and every prior phase landed on main).

### Gotchas
- **Mint an agent JWT with `create_access_token(6)`** — the function takes a **user_id int** (id 6 =
  agent user), *not* a `{'sub': ...}` dict. Passing a dict yields a 160-char token that 401s. Also
  the Bearer header needs the space: `-H "Authorization: Bearer $TOKEN"`.
- The api no-bind-mount gotcha did **not** bite — the backend was already rebuilt last session, so
  all `/bins` + `/chests` + `/presets` routes were live in the running container (confirmed via
