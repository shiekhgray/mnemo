### Key decisions
- **Read geometry from the photos rather than guess.** The whole point of the visual finder is
  fidelity, so fabricating drawer geometry would defeat it. Confirmed 6/8 cabinets confidently
  from the close-ups; flagged the angled-photo ones for Graham, who corrected them.
- **Renumbering reshuffles existing numbers** (old Cabinet 3 → Cabinet 4, old 5 → 7, etc.).
  Surfaced this consequence and remapped the stored part locations in the same pass so nothing
  was left pointing at a stale name. Did the remap as a single snapshot pass with a temp-code
  two-pass on bin codes to dodge the `UNIQUE(code)` constraint mid-rename.
- **In-place relabel, not a re-seed, for the rename** — no slot churn, so Cabinet 1's
  assignments were untouched.

### Gotchas
- **The `api` container is NOT bind-mounted** — Python is baked into the image. `restart api`
  runs stale code; you must `docker compose up -d --build api` to pick up backend/script edits.
  Cost a confused round (search results missing `location_ref`, then a reseed script importing
  the pre-edit `WALL_LAYOUT`). Now documented in CLAUDE.md.
- Playwright in `web/node_modules` is CommonJS — `import pw from '…/playwright/index.js'; const
  {chromium}=pw`, not a named import.
- ESLint forbids `setState` inside an effect — seed teleport state via `useState` initializers
  and use the effect only to strip the URL params.
